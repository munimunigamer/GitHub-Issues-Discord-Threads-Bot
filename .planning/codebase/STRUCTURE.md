## Naming Conventions

**Files:**
- Lowercase with camelCase: `discordHandlers.ts`, `githubActions.ts`
- Domain-based grouping: Files in `src/discord/` handle Discord concerns, `src/github/` handle GitHub
- Suffixes: `Handlers.ts` (event handlers), `Actions.ts` (API operations)

**Directories:**
- Feature/domain grouped: `discord/`, `github/`
- Lowercase: `src/`, `dist/`, `node_modules/`

**Functions:**
- camelCase: `handleThreadCreate()`, `createIssue()`, `getThreadChannel()`
- Verb prefix for actions: `create*`, `archive*`, `close*`, `delete*`, `lock*`, `unlock*`
- Handler prefix for event handlers: `handle*` (handleClientReady, handleThreadCreate)

**Variables:**
- camelCase for normal variables: `thread`, `appliedTags`, `channelId`
- Uppercase for constants/enums: `Actions`, `Triggerer`, `DISCORD_TOKEN`
- Descriptive names with context: `threadPromises`, `availableTags`, `repoCredentials`

**Types:**
- PascalCase for interfaces: `Thread`, `ThreadComment`, `GitIssue`, `GitHubLabel`
- Type suffix for type aliases: `GithubHandlerFunction`

## Where to Add New Code

**New Discord Event Handler:**

1. Add handler function to `src/discord/discordHandlers.ts`:
```typescript
export async function handleNewEvent(params: AnyThreadChannel) {
  if (params.parentId !== config.DISCORD_CHANNEL_ID) return;
  // ... handler logic
}
```

2. Register listener in `src/discord/discord.ts`:
```typescript
client.on(Events.NewEvent, handleNewEvent);
```

3. Add corresponding action function(s) to `src/discord/discordActions.ts` if needed

**New GitHub Webhook Handler:**

1. Add handler function to `src/github/githubHandlers.ts`:
```typescript
export async function handleNewAction(req: Request) {
  // ... handler logic
}
```

2. Register in `src/github/github.ts` githubActions map:
```typescript
const githubActions: {
  [key: string]: GithubHandlerFunction;
} = {
  newaction: (req) => handleNewAction(req),
  // ...
};
```

**New Utility/Helper:**

- Shared utilities: Add to `src/logger.ts` (for logging/tracking) or create new module at `src/utils.ts`
- Domain-specific helpers: Add to respective domain module (e.g., `src/github/githubActions.ts` for GitHub helpers)

**New Integration:**

1. Add credentials to `src/config.ts` with validation
2. Create new module in appropriate domain directory
3. Initialize in corresponding init function (initDiscord or initGithub)
4. Export singleton instance for use throughout codebase
